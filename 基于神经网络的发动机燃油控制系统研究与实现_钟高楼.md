导师: 姚凯学

# 摘要

随着航空测控技术的不断进步，对飞机燃油控制系统的全面测控需要发动机等部件进行联动；如果使用发动机等部件来搭建的试验台造价非常昂贵，因此，为节省成本可以将发动机等相关部件采用模型虚拟仿真的方法参与进行测试，形成半物理试验台。本文以飞机燃油系统半物理试验台为基础，设计飞机燃油半物理试验台测控系统。其主要工作如下：

(1) 针对测控系统在进行数据采集时，存在系统的实时性和响应速度下降、数据传输速度慢等问题，在 DMA 的基础上引入多缓冲区技术和链表化任务管理策略优化 DMA 传输效率，提高数据采集的实时性。测试结果表明，测试结果表明，优化后的 DMA 方案提升了数据传输速率。

(2) 针对试验台在测试时对加力分布器出口燃油流量测量复杂及准确度不高的问题，提出了一种结合卷积神经网络（CNN）与长短期记忆网络（LSTM）的深度学习方法，以实现对燃油流量的预测。实验结果显示，该模型能够有效预测加力分布器出口的燃油流量，精度达到预期要求。

(3) 设计并实现了一套试验台测控系统，该系统分为数据采集系统和控制系统。数据采集系统基于“生产者/消费者”模式，采用多模块多进程架构，通过数据采集板卡对试验台设备的状态数据进行采集，包括电机转速、阀门位置和燃油压力等关键参数。控制系统采用三层架构，包括表示层、逻辑层和数据层。系统通过接口实现与主燃油系统数学模型和发动机数学模型的数据交换，并针对试验台的特点设计了控制系统与辅助控制系统之间的通信结构和协议。此外，系统采用内存反射卡与光纤相结合的方式进行数据交换，以满足实时性要求。

**关键字：**测控系统；数据采集；DMA；设备控制

# ABSTRACT

With the continuous advancement in aviation monitoring and control technology, comprehensive monitoring and control of aircraft fuel control systems necessitate the integration of components such as engines. However, constructing a test bench using actual engines and related components is prohibitively expensive. To reduce costs, a cost-effective alternative is to utilize virtual simulation models of these components to participate in testing, thus forming a semi-physical test bench. This paper designs a monitoring and control system for an aircraft fuel semi-physical test bench, with the main contributions as follows:

(1) In response to issues such as decreased real-time performance and response speed of the monitoring and control system during data acquisition, as well as slow data transmission speed, a multi-buffering technique and a linked-list task management strategy were introduced based on Direct Memory Access (DMA) to optimize DMA transmission efficiency and improve the real-time performance of data acquisition. Test results indicate that the optimized DMA scheme significantly enhances data transmission rates.

(2) To address the complexity and low accuracy of measuring the fuel flow rate at the afterburner distributor's outlet during testing, a deep learning method combining Convolutional Neural Networks (CNN) and Long Short-Term Memory (LSTM) networks was proposed to predict the fuel flow rate. Experimental results demonstrate that the model effectively predicts the fuel flow rate at the afterburner distributor's outlet, meeting the expected accuracy requirements.

(3) A test bench monitoring and control system was designed and implemented, consisting of a data acquisition system and a control system. The data acquisition system, based on the "producer/consumer" model, employs a multi-module, multi-process architecture to collect state data from test bench equipment through data acquisition boards, including key parameters such as motor speed, valve position, and fuel pressure. The control system adopts a three-layer architecture, comprising the

presentation layer, logic layer, and data layer. The system facilitates data exchange with the main fuel system's mathematical model and the engine's mathematical model through interfaces, and a communication structure and protocol were designed between the control system and auxiliary control system, tailored to the characteristics of the test bench. Furthermore, the system integrates memory-reflective cards and optical fibers for data exchange to meet real-time requirements.

**Key words:** Measurement and Control System; Data Acquisition; DMA; Device Control

# 第一章 绪论

## 1.1 研究背景和意义

飞机燃油系统是飞机正常飞行的重要保障，其能够保证在任何规定的状态下，向发动机持续不间断的提供符合技术标准的燃油<sup>[1,2]</sup>。加力燃油泵、加力调节器、加力分布器是飞机燃油系统的核心部件，决定着所供应燃油的压力和流量是否符合技术要求。因此，在飞机燃油系统的核心部件的设计与生产过程中，需要对产品进行各项运行试验，以准确掌握其各项性能参数<sup>[3]</sup>。

目前，对飞机燃油系统核心部件的性能测试主要依靠飞机燃油系统试验台进行。搭建飞机燃油系统核心部件的运行环境，对飞机燃油系统核心部件进行测试，并通过测控系统对试验台设备和加力泵、加力调节器等被试件进行数据采集和控制<sup>[4-6]</sup>。而飞机燃油系统半物理试验台能否严格按照技术标准进行试验，则取决于试验台的计算机测控系统。同时，计算机测控系统还需要对测试台的燃油供油设备、被试件控制器、故障注入设备进行控制和调节。

当下，我国航空产业进入飞速发展时期，各种新型号飞机不断设计投产，依照国家标准，飞机燃油系统制造厂家在产品出厂前必须对其进行性能试验，检测合格后才能进行交付，以确保产品不会造成飞机飞行安全问题。因此，企业需要搭建相应的试验台以实现对飞机燃油系统主要部件性能进行测试和验证。如果使用发动机等部件搭建飞机燃油系统核心部件的试验台，会使得试验台造价成本高。飞机燃油系统产品专业性强、技术细节保密性高，因此，企业使用由厂家提供的数学模型来代替发动机和相关部件参与飞机燃油系统核心部件的测试，这样既达到对飞机燃油系统核心部件进行系统测试目的，又节约测试成本。

在目前的实际生产中，厂家自行设计的飞机燃油系统半物理试验台测控系统仍存在着一些问题未能有效解决，如手工操作居多、自动化程度低、功能单一、结构复杂、对试验水平要求较高等<sup>[7-9]</sup>，这些问题不仅影响了试验的效率，还容易引入人为错误，导致试验结果不准确。此外，现有的试验台测控系统还

存在故障率高、故障排除难度大等问题，这在一定程度上阻碍了加力泵等的生产和检测。

针对现有的试验台测控系统存在的问题，诸多企业对飞机燃油系统半物理试验台测控系统展开了大量的研究。随着自动化控制技术和测控技术<sup>[10-12]</sup>的不断进步，虚拟仪器和半物理试验台的提出为解决上述问题提供了全新的途径。

在上述研究的基础上，针对现有的试验台测控系统存在的问题并结合厂家实际需求，本文采用自动化控制技术、虚拟仪器等技术<sup>[13-15]</sup>，融合了硬件驱动、软件控制和模型仿真，设计了一个飞机燃油系统试验台测控系统。在实现对试验台设备状态采集的基础上，并能够有效的对试验台设备进行控制，实现对被试件的自动化测试。

因此，本文设计的测控系统在试验台进行被试件测试展现出一定的应用价值，为加力泵等被试件的性能提升和可靠性保障的研究提供了技术支持，为测试系统的开发提供参考。

## 1.2 国内外研究现状

随着计算机技术的发展，工业生产中对测控技术的要求变得更加严格和细致。传统的试验台测控系统因其复杂性和高成本，以及完全依赖人工操作导致的数据误差和结果不一致性。相比之下，现代测控技术借助计算机技术的进步，已经发展成为更高效、准确的系统。它通过网络化实现了数据采集、资源共享、实时监控和精确控制，极大提升了测试效率。这种技术的应用不仅简化了操作，降低了成本，提高了测控结果的准确性和稳定性。

2014 年，Li 等人提出了一种结合测量和控制的技术<sup>[16]</sup>。利用计算机集成、多线程技术的液压支架实验台的自动控制系统。该系统基于数据采集，分别设置测量和控制模块，实现了模拟数字转换和数字模拟/数字输出控制的功能。经过实际运行测试后得出，系统的响应、测量和控制精度得到了提升。

2015 年，杨等人设计了一套基于 LabVIEW 的轴承试验机测试与控制系统<sup>[17]</sup>。该系统能够利用虚拟仪器技术实现对试验机各项性能指标的监测，并且该系统能够精确控制试验机的主轴转速、加载载荷以及润滑油温度等关键参数，确保试验机按照设定的状态高效稳定运行。较常规的测控系统有很大的改进和

提升，提高了系统工作效率，保证了控制的可靠性。在分析了影响轴承性能的主要原因的基础上，提出了一种基于 PID 的简单的负载调节方法，该方法可以获得较高的精度，但是不能满足系统的快速响应要求。

2015 年，Wang 等人设计了发动机实验数据采集系统，完成系统硬件的搭建，并根据不同的传感器完成传感器处理电路的设计<sup>[18]</sup>。并基于 VC++平台开发了发动机数据采集系统的软件。该系统可以减少繁重的手工记录，并确保实验的准确性和有效性。

2016 年，陈等人设计了一种以工业计算机和 PCI 数据采集卡为核心，LabWindows/CVI 为软件开发平台的液压泵试验台测控系统<sup>[19]</sup>。该系统利用滞环控制算法将油温控制在 35~45°C 之间，利用 PID 算法实现出油口、回油口压力控制。该系统存在局限性，只能完成某型产品的测试，并且该系统无法进行功能扩展。

2017 年，沙等开发了一套液压支架试验台远程实时测控系统<sup>[20]</sup>，该系统将 PXI 平台应用于现场级硬件开发，数据通过光纤以太网传输至集中管理控制室，大大简化了现场与控制室的电缆布线，并且该系统具有较高实时性和控制精度，大大提高了液压支架试验台的自动化水平和检验效率。

2017 年，Ma 等人<sup>[21]</sup>针对现有水泥企业数据采集系统中串口数据资源利用率低的问题，基于 C#多线程技术，以串口通信为纽带，实现了多串口下的仪表数据采集。该系统显示了数据收集的结果，提高了计算机内存的利用率，加强了数据采集的可靠性和准确性，为水泥企业的数据应用提供了准确、可靠的数据支持。

2017 年，梁设计实现了一种基于 LabVIEW 的泵出厂试验台测控系统<sup>[22]</sup>，该系统采用了以工控机和 PLC 相结合的结构设计，可以有效的提高被试泵的检测速度。然而该系统采用 RS232 通信，该通信方式传输距离近，无法组网，使测控系统不能使用现有的端口接入其他设备。

2018 年，Su 等人针对船用部件的机械性能测试具有结构复杂、测试资源多样、测试环境多变、测试成本高和周期长的特点提出了一种集中控制与分布控制相结合的测控系统框架<sup>[23]</sup>。该系统框架在考虑测试平台非线性因素的基础上建立伺服控制系统的传递函数。并设计开发了测试平台分布式测量与控制系统

的硬件和软件。经过 2000KN 拖缆拉伸实验验证，系统的力量测量误差小于  $\pm 0.5\%$ ，位移测量误差小于  $\pm 0.1\%$ 。

2018 年，郝设计实现了一种油泵试验台测控系统<sup>[24]</sup>，该系统使用 C8051F020 单片机完成试验台各个模块的数据采集处理与控制。该测控系统喷油量的测试误差在 1%左右，油箱温度测量精度为  $0.1^\circ\text{C}$ ，还解决了试验台效率低、人工参与多的问题。

2018 年，贾研究与设计了一套以工业计算机为核心的综合试验台测控系统<sup>[25]</sup>，该系统利用 NI 板卡实现对信号的采集和控制，由 ACS880 变频器和 PLC 对试验台动力系统和辅助控制系统进行控制，并在 LabVIEW 的基础上，对软件进行了编程，既可以进行手工测试，也可以进行自动化控制与测试，还可以进行多个位置的同步检测。然而该系统的变频干扰使弱电信号中夹杂着较大的随机噪声，无法满足系统对精度的要求。

2018 年，Ridwan 等人提出一种结合 LabVIEW 作为仿真平台的通用软件无线电外设的可重配置 FiWi 测试平台<sup>[26]</sup>。该测试平台，测试了无线范围和功率输出，优化 USRP 的最佳输出。该研究还评估了上行 FiWi 测试平台传输的性能，包括吞吐量、传输时间和抖动。

2020 年，孟等人针对传统现场总线应用在现代测控系统领域存在的数据传输实时性差、系统扩展复杂的问题，在 EtherCAT 总线的基础上，设计了一种分布式测控系统<sup>[27]</sup>。采用 EtherCAT 总线的方式实现了分布式测试系统的实时、可靠、易于扩充。将 EtherCAT 总线技术应用到分布式测控系统中，解决了传统的现场总线传输实时性差和系统扩展复杂等问题，提高了整个系统数据传输的实时性和可靠性，且便于对系统结构进行扩展。

2020 年，Muna 等人提出了一个新的通用的端到端 IIoT 安全测试平台，并提供了测试平台的架构设计和实施过程的详细信息<sup>[28]</sup>。所提出的测试平台可以轻松复制和重新配置，以支持新流程和各种安全场景的测试活动。

2021 年，刘等人研制了一种针对二维泵试验台设计的测控系统<sup>[29]</sup>。该系统采用了工业控制计算机作为上位机，PLC 作为下位机，实现对试验台的自动控制。通过该系统，可以实现对二维泵试验台各项参数的实时监测和控制，确保试验过程的稳定性和准确性。

## 2.1 直接存储器访问

DMA (Direct Memory Access, 直接存储器访问) 是一种计算机系统中的数据传输技术, 它允许外设设备直接与系统内存进行数据交换, 而不需要经过中央处理器 (CPU) 的干预。这种机制显著提高了数据传输效率, 特别是在需要处理大量数据的场景中, 如音频、视频流处理和高速数据采集系统等。

在传统的数据传输过程中, CPU 通常需要参与数据的读取和写入操作, 这不仅占用了 CPU 的处理时间, 还可能导致系统性能的瓶颈。DMA 通过引入一个独立的 DMA 控制器 (DMAC) 来管理数据传输过程, 从而解放了 CPU 的资源。具体来说, DMA 传输可以分为以下几个步骤:

(1) DMA 请求: 外设向 DMA 控制器发出数据传输请求。

(2) DMA 控制器配置: DMA 控制器接受请求后, 配置源地址、目标地址和传输数据的大小等参数。

(3) 总线掌控权移交: DMA 控制器获取总线控制权, 暂时接管 CPU 与内存之间的数据传输路径。

(4) 数据传输: DMA 控制器将数据从源地址直接传输到目标地址。此过程中, CPU 可以继续执行其他任务, 直至数据传输完成。

(5) 传输完成通知: DMA 控制器完成数据传输后, 通过中断信号通知 CPU, 标志传输结束。

## 2.3 长短期记忆神经网络

长短期记忆神经网络（Long Short Term Memory, LSTM）是一种特殊的循环神经网络（Recurrent Neural Network, RNN），专门用于解决传统 RNN 在处理时序数据时面临的长期依赖问题<sup>[35]</sup>。LSTM 神经元的结构如图 2-4 所示。

LSTM 包括三个门结构，分别是遗忘门、输入门和输出门。因其复杂的门控结构，LSTM 神经元间的拓扑结构与 RNN 有较大差异。与传统的循环单元覆盖前一时间步信息的做法不同，LSTM 单元通过引入门控单元来决定是否维持上一时间步的信息。若 LSTM 单元在较早的时刻从输入序列中捕获到关键特征，可通过门控单元进行保留，从而使神经网络能够轻松地将此特征信息传递至后续时刻。对于非关键特征，LSTM 则通过遗忘门进行淘汰。

图 2-4 LSTM 神经元结构

遗忘门决定了哪些信息应该从单元状态中被遗忘。它通过接受当前输入  $x_t$  和前一个时间步的隐藏状态  $h_{t-1}$ ，与 Sigmoid 函数相乘后输出一个 0 到 1 之间的值给每个在单元状态中的数值  $f_t$ ，当  $f_t$  中的数值为 1 表示“完全保留”，为 0 表示“完全丢弃”。遗忘门的计算如式(2-1)所示。

$$f_t = \sigma(W_u \cdot [h_{t-1}, x_t] + b_u) \quad (2-1)$$

式中  $\sigma$  表示 Sigmoid 激活函数； $f_t$  用于控制过去信息量； $W_u$  是输入和隐藏状态之间的权重矩阵； $h_{t-1}$  是前一个时间步的隐藏状态； $x_t$  是当前时间步的输入； $b_u$  是偏置项。

输入门决定哪些新信息将被添加到单元状态中。输入门接受  $x_t$  和  $h_{t-1}$  作为输入矩阵，分别与各自的输入门权重做矩阵乘法后求和，然后分别经过 Tanh 函数激活和 Sigmoid 函数激活，得到输入门的输入矩阵的中间状态  $\tilde{C}_t$  和门控矩阵  $i_t$ 。最后  $i_t$  与  $\tilde{C}_t$  做 Hadamard 乘积，即是对当前时刻的信息进行选择，重要的信息保留并输入到细胞状态矩阵中。上一时刻保留到当前时刻的细胞状态矩阵和当前时刻流向下一时刻的信息矩阵求和后，得到当前时刻神经元的细胞状态  $C_t$ 。输入门计算公式如式(2-2)、(2-3)和(2-4)所示。

$$i_t = \sigma(W_i \cdot [h_{t-1}, x_t] + b_i) \quad (2-2)$$

$$\tilde{C}_t = \tanh(W_i \cdot [h_{t-1}, x_t] + b_i) \quad (2-3)$$

$$C_t = i_t \odot \tilde{C}_t + f_t \odot C_{t-1} \quad (2-4)$$

式中， $i_t$  用于控制输入信息量； $W_i$  是输入和隐藏状态之间的权重矩阵； $h_{t-1}$  是上一个时间步的隐藏状态； $x_t$  是当前时间步的输入； $b_i$  是偏置项； $\tilde{C}_t$  为中间状态； $C_{t-1}$  为过去的信息； $C_t$  为新的输入信息； $\odot$  表示 Hadamard 乘积。

输出门决定当前时间步的隐藏状态与单元状态之间的映射关系，以及当前时间步的隐藏状态如何影响输出，输出门通过一个 Sigmoid 函数来输出一个介于 0 到 1 之间的值，表示单元状态中有多少信息应该被输出，输出门的计算公式如式(2-5)、(2-6)所示。

$$o_t = \sigma(W_o \cdot [h_{t-1}, x_t] + b_o) \quad (2-5)$$

$$h_t = o_t \odot \tanh(C_t) \quad (2-6)$$

式中， $o_t$  用于控制信息量； $W_o$  是输入和隐藏状态之间的权重矩阵； $h_{t-1}$  是上一个时间步的隐藏状态； $x_t$  是当前时间步的输入； $b_o$  是偏置项； $h_t$  可以看作  $C_t$  的一部分，对  $C_t$  有选择的输出。

三个门控单元共同作用于 LSTM 模型中的单元状态，通过控制输入、遗忘和输出的信息流动，使模型捕捉和学习序列数据中的长期依赖关系，提高模型性能。

# 第三章 基于 DMA 的实时数据采集方法研究

## 3.1 问题描述

各类数据采集系统在工业控制、环境监测、医疗诊断等领域的应用日益广泛，这些系统需要采集来自多种传感器的大量数据，并对数据进行实时的分析与处理。然而，随着数据量和采集频率的不断增长，传统的数据传输方法在效率和实时性方面面临着严重的挑战。

传统的数据传输方法主要依赖于中断驱动或轮询方式。这些方式在低速、低数据量的场景下能够有效工作，但在面对大规模、高速的数据传输需求时却暴露出显著的局限性。中断驱动方式频繁占用 CPU 资源，导致系统的实时性和响应速度下降；轮询方式则要求 CPU 不断查询设备状态，造成 CPU 的工作负载增加，系统整体效率降低。此外，这两种方式都要求 CPU 直接参与数据的传输过程，无法充分利用系统硬件资源，进而限制了数据采集系统的性能。

针对这些问题，DMA（直接存储器访问）技术被广泛应用于数据采集系统中。DMA 技术能够在不依赖 CPU 的情况下直接完成数据在存储器与外围设备之间的传输，从而减少 CPU 负担，提高系统的整体效率和实时性。DMA 数据传输如图 3-1 所示。

然而，传统的DMA技术在应对当前复杂多样的数据采集需求时，仍然存在一定的优化空间。首先，传统DMA控制器在处理多任务数据传输时，容易出现资源冲突和传输延迟，尤其是在面对多个高频率数据流时，系统的吞吐量可能受到影响。其次，由于不同传感器的数据采集频率和数据量各不相同，如何合理地调度和管理DMA任务，以充分利用硬件资源，实现数据传输的高效性和稳定性，成为了系统设计中的关键问题。

## 3.2 多缓冲区技术

在实时数据采集系统中，DMA数据传输的延迟和不稳定性是影响系统性能的主要因素之一。传统的单缓冲区DMA传输方式通常要求数据采集与数据传输在同一缓冲区中进行，这种方式在采集和传输过程中容易产生冲突，导致系统性能下降，尤其是在高频数据采集场景中，DMA数据传输的延迟问题更加明显。

为了解决上述问题，本文在DMA的基础上引入了多缓冲区技术。这种技术通过同时使用多个缓冲区，实现数据采集与数据传输的并行操作，减少了数据在传输过程中的等待时间，从而提高了系统的整体效率。

系统将数据分配到两个或多个缓冲区中进行交替使用。当一个缓冲区的数据正在通过DMA传输时，另一个缓冲区可以同时进行新数据的采集或处理。一旦一个缓冲区的数据传输完成，系统立即切换到下一个缓冲区进行传输，而前一个缓冲区则开始准备下一组数据。这种交替使用的方式确保了数据传输和处理的连续性，避免了传统单缓冲区方法中由于等待导致的性能瓶颈。多缓冲区技术如图 3-2 所示。

## 3.4 DMA 功能设计

### 3.4.1 DMA 功能结构设计

考虑到不同采集板卡对不同传感器采集频率的差异，数据采集系统中的数据传输效率和实时性显得尤为重要。为了解决这一问题，系统采用了直接存储器访问（Direct Memory Access, DMA）技术，使得大量数据能够在不依赖 CPU 的情况下直接传输到系统存储器中<sup>[44-46]</sup>。这种方式不仅释放了 CPU 资源，降低了系统中断和轮询带来的负担，还显著提高了数据采集的效率。

在典型的数据采集系统中，各类传感器通过采集板卡将数据传输至系统，一般采用中断或者轮询的方式进行传输，而这些数据往往具有不同的采集频率和数据量，这样就造成数据采集效率低，数据传输延时大等问题。采用 DMA 技术，外围设备在准备好数据后，只需向 DMA 控制器发送一个启动命令，传递数据的存储地址和数据大小信息，DMA 控制器便会接管后续的传输过程，直接将数据从外围设备传输至内存中。这一机制极大地简化了数据传输流程，并确保数据能够及时存储，满足实时处理需求。

为了进一步提升系统的传输效率，DMA 控制器通常工作在突发传输模式（Burst Mode）下。在这一模式中，DMA 控制器可以一次性传输多个字节的数据，而无需在每次传输操作之间等待 CPU 的干预。这种高效的数据传输方式不仅提高了数据传输速率，还减少了系统总线的占用时间，从而大幅增强了数据采集系统的整体吞吐能力。与传统的中断驱动或轮询方式相比，DMA 技术在减轻 CPU 负担的同时，还能显著增加数据吞吐量，最终提升整个系统的性能。

对于本文所设计的数据采集系统，不仅需要将从采集板卡获取的数据传输至内存，还需要将经过处理的数据再传输到其他存储设备或反射卡上。在这种情况下，DMA 控制器需要在多个存储区域之间执行高速的数据传输任务。为了有效管理和调度这些传输任务，本文设计一种基于链表的 DMA 任务调度策略。

具体而言，多个 DMA 任务被以链表的形式链接起来，每个任务描述符（descriptor）包含了该任务的数据源地址、目标地址、传输数据量及其他必要的控制信息。在 DMA 传输过程中，DMA 控制器会依次读取链表中的任务描述符，并按照描述符中的信息执行相应的传输操作。通过这种链式结构，不仅能够有效减少 CPU 对传输任务的管理开销，还能在不间断的传输中提高系统的并行处理能力，确保数据在不同存储区域之间的高效传输。通过将 DMA 任务以链表结构组织起来，系统可以灵活地处理各种数据传输需求，避免了传统单一任务处理模式的瓶颈，进一步优化了数据采集系统的整体性能。DMA 传输结构如所示。

### 3.4.2 DMA 功能配置

DMA 从采集板卡将传感器数据传输到内存时，需要提前在内存中为存放传感器数据开辟物理存储空间，考虑到 DMA 传输速度和 CPU 读取速度不一致的

情况，为了提高 DMA 的传输效率，采用多缓冲区技术，在 DMA 传输数据前，开辟两个同样大小的物理存储空间，DMA 传输数据时，将数据分配到两个缓冲区中交替使用。当物理存储空间 A 的数据正在通过 DMA 传输时，物理存储空间 B 可以同时进行数据的写入或读取操作。一旦物理存储空间 A 的数据传输完成，立即切换到物理存储空间 B 进行传输，而物理存储空间 A 则准备下一组数据。这样做的好处是减少了等待时间，实现了传输和数据处理的并行。

本文利用 PXI 设备中的 DMA 控制器、并通过软件开辟物理内存以及配置 IP 核的方法，实现了在 PXI 总线架构下的 DMA 高速数据传输。首先需要获取 DMA 传输的配置信息。通过调用 DMAInit 函数获取对物理内存进行操作所需的指针。随后，利用 DMAMemAlloc 函数依据所采集数据大小的需求开辟 2 份等长连续的物理内存区域，同时记录下每段内存区域的起始地址。这两段物理内存区域作为上面提到的双缓存技术所使用的两个缓冲区。然后根据需要获取 DMA 传输任务 1 和 2 的其他必要控制信息，如数据传输大小、传输模式等等，并将这些控制信息保存到链表中。DMA 控制器可以根据链表中的控制信息完成从采集板卡到内存，然后从内存到内存反射卡的数据传输工作。

对于 DMA 传输任务 1，基于在 DMA 配置阶段所设定的物理内存长度，当采集板卡在上次 DMA 传输周期后新采集的数据长度达到每份物理内存区域预设的长度时，固件逻辑将自动判定该条件成立，并启动 DMA 控制器进行数据传输。数据传输完成后，DMA 控制器将向主机发送 DMA 完成中断信号。在 DMA 完成中断的处理函数中，所需执行的操作仅限于将对应物理内存区域中的数据写入文件，并重置 DMA 控制器的写地址寄存器，以为下一次的数据传输做好准备。

对于 DMA 传输任务 2，按照配置好的 DMA 配置信息，将处理好的传感器数据从内存写入到内存反射卡中，与控制系统进行数据交换。

# 第四章 基于 CNN-LSTM 的燃油流量预测方法研究

## 4.1 问题描述

在试验台上进行被试件测试时，测量加力分布器出口燃油流量的方法往往相对复杂，主要原因在于试验环境中存在多种不可控因素，这些因素会对测得的燃油流量数据产生显著干扰，进而降低测量的精度。具体而言，这些干扰因素包括但不限于传感器噪声、环境温度的波动、振动的影响等。这些因素通常具有动态性和不可预测性，传统的流量测量方法因此难以确保测量结果的稳定性和可靠性。

为提高燃油流量数据的精度，传统方法通常采用多点校准技术以及复杂的滤波算法。然而，这些方法在实际应用中面临诸多挑战。首先，校准和滤波过程通常需要大量的实验数据和复杂的计算步骤，增加了实验的操作难度和成本。其次，在面对非线性和高度耦合的干扰因素时，传统方法的有效性往往受到限制，尤其是在高动态范围和复杂工况下，现有的测量和校正技术难以有效应对这些复杂条件，导致测量结果的精度难以保障。

本文提出了一种结合卷积神经网络（CNN）与长短期记忆网络（LSTM）的深度学习方法，以实现对燃油流量的精确预测。具体来说，CNN 通过多尺度卷积操作，有效提取与加力分布器出口燃油流量高度相关数据中的时空特征，并捕捉复杂工况下流量变化的时空模式。与此同时，LSTM 网络凭借其对序列数据中长期依赖关系的记忆能力，能够处理燃油流量数据中的时序依赖性和长期趋势。通过将 CNN 与 LSTM 相结合，构建一个在复杂工况下依然具有高度精确性的燃油流量预测模型，从而提升燃油流量测量的可靠性与准确度。

在本文中，所使用的数据集来源于试验台在测试阶段通过传感器以每秒一次的频率采集的传感器数据。为了确保模型的泛化能力和测试的可靠性，将整个数据集按照 75%和 25%的比例划分为训练集和测试集。在数据预处理阶段，首先剔除了数据集中的异常值，这些异常值可能是由于传感器故障或其他外部干扰因素所引起的。在处理少量的缺失值时，采用了基于缺失值前后两条记录中相同特征的平均值进行插补的方法，以尽可能保留数据的完整性和一致性。经过这些预处理步骤后，最终得到的数据量约为 11000 条。

值得注意的是，不同数据特征的维度差异可能对机器学习模型的训练过程产生不利影响。具体来说，这种维度差异可能会导致模型在训练过程中忽略那

些特征值较小的维度，从而过于依赖特征值较大的维度，最终影响到模型的预测性能和决策判断。为了解决这一问题，在模型训练之前，对数据进行了归一化处理。归一化的目的是将不同特征的数值范围缩放到相同的尺度，以消除维度差异对模型训练的不利影响。常见的归一化方法包括 Z-score 标准化和 Min-Max 归一化。这两种方法都能够有效地将数据转换到同一尺度上，从而确保模型对各个特征的平等关注，并有助于提高模型的训练效果和预测准确性。

### 4.3.2 模型评价指标

为验证模型性能，本文使用均方根误差（RMSE）、平均绝对误差（MAE）和拟合优度（ $R^2$ ）作为模型性能的评价指标。

RMSE 的计算公式如式(3-1)所示。

$$RMSE = \sqrt{\frac{1}{n} \sum_{i=1}^{n} (y_i - \hat{y}_i)^2} \quad (3-1)$$

MAE 的计算公式如式(3-2)所示：

$$MAE = \frac{1}{n} \sum_{i=1}^{n} |y_i - \hat{y}_i| \quad (3-2)$$

$R^2$  的计算公式如式(3-3)所示，其用于衡量模型的预测能力，范围在[0,1]之间变动，越接近 1 则表明模型对观测数据的拟合程度越好：

$$R^2 = 1 - \frac{\sum_{i=1}^{n} (y_i - \hat{y}_i)^2}{\sum_{i=1}^{n} (y_i - \bar{y})^2} \quad (3-3)$$

式(3-1)到式(3-3)中， $y_i$  表示实际值， $\hat{y}_i$  表示预测值， $\bar{y}$  为预测值均值， $n$  表示样本数量。

## 5.1 测控系统需求分析

需求分析是系统设计的关键步骤，尤其对于测控系统这样的复杂系统更是如此。在测控系统的设计过程中，通过需求分析识别系统的功能需求，发现潜在的问题，从而提前解决，并确保系统的可靠性和高效性。

### 5.1.1 系统功能需求分析

本系统根据企业生产工作方面的需要，对测控系统的设计需求进行了深入的分析。在数据采集、数据传输、数据处理、设备控制、人机交互等方面，不同的业务功能需求涵盖了不同角色的实际需求和工作需要。基于测控系统各模块的业务流程，本文设计的试验台测控系统具有以下的功能需求：

(1) 数据采集与处理：系统需要具备试验台状态数据的采集功能，能够连续地收集试验过程中的所有数据，并进行必要的预处理，例如滤波、去噪等，为后续的数据分析提供数据源。

(2) 实时监测与控制：试验台测控系统为了确保试验过程的精确性和安全性，系统需要具备监测试验台设备及被试件的工作状态的能力，并能根据实验要求对相关参数进行控制。

(3) 数据储存与检索：对于试验中获取的大量数据，系统需要进行妥善的存储，确保数据的安全性，同时方便后续的检索与分析。此外，存储的数据还应该便于导出和分享。

(4) 集成化与自动化：为了提高试验效率，系统应能与其他相关设备或系统进行集成，实现信息的共享和协同工作。此外，系统还应具备一定的自动化功能，如自动校准、自动检测等，减少人工干预。



### 5.1.2 系统非功能需求分析

非功能性需求分析是对产品或系统的非功能特性进行评估的过程，涉及可靠性、安全性、可扩展性等方面。对于试验台测控系统而言，需重点关注以下非功能性需求：

(1) 性能需求：系统应在正常工作负载下实现低延迟和高速数据处理，以确保能够迅速响应操作请求，并保持对大规模试验数据的实时处理与准确性。

(2) 可靠性需求：测控系统须展现出极高的可靠性与稳定性，最小化因故障导致的系统停机时间，并在遭遇硬件故障等异常情况时，仍能在满足安全机制前提下维持运行，影响最小化。

(3) 安全性需求：系统应实施严格的身份验证流程，确保仅授权用户访问，依据用户角色执行访问控制，并对所有传输与存储的敏感数据执行加密处理。

(4) 可维护性需求：系统需记录细致的操作和事件日志，以便于故障诊断和安全审计，并提供简便的软件升级机制，以适配新的测试需求和功能升级。

(5) 可扩展性需求：测控系统应兼容多种硬件配置，并采纳模块化设计原则，以支持硬件的未来升级和扩展，以及新功能模块和传感器的加入。

通过满足以上非功能性需求，试验台测控系统能够在保障高效性、稳定性和安全性的同时，提供良好的用户体验，适应未来的技术进步和扩展需求。

## 5.3 测控系统总体架构设计

试验台测控系统为双机架构，以数据采集计算机作为下位机，控制计算机作为上位机。数据采集系统进行试验台数据采集和控制，控制系统承担人机交互和过程控制及数据处理等功能。

数据采集系统记录试验台在测试过程中的试验台设备状态、控制命令等全部数据。数据采集系统需要对多个采集控制板卡的多个通道进行采样，并且可以通过调整采样频率设置不同的数据采集速率，实现对试验台状态进行全面的监测。因此随着采集数据类型和通道数量的增加，总体采集时间增长，进而造成了数据采集系统各个采集通道之间的时钟同步问题。此外，采集卡内部缓冲区大小、数据传输速率以及系统负载等因素亦对时钟同步产生显著影响<sup>[39-41]</sup>。因此本文在多通道数据采集系统中，采用不同类型的采集卡负责采集特定类型的信号，通过轮询方式逐个访问通道以完成数据采集任务。

同时，为了减少各板卡之间的时钟误差，确保各采集卡之间的时钟同步和数据传输同步。利用多个采集卡同时采集信号，并同步处理数据。关键在于确保各采集卡时钟同步和数据传输同步。在高速数据采集系统中，微小的时钟偏差可能导致严重的数据误差。一种常见做法是使用高精度时钟发生器作为外部

时钟源，为所有采集卡提供统一时钟信号。同时，结合硬件和软件实现各采集卡间数据传输同步，确保数据准确性和一致性。

### (3) 内存反射卡

本系统为了实现高速、低延迟和可靠数据通信，采用内存反射卡实现不同系统之间的数据共享和同步。当数据采集系统向内存反射卡写入试验台设备状态时，控制计算机和模型运行计算机能够同步获取数据采集系统写入的试验台设备状态。内存反射卡的工作原理基于内存映射技术，即将一块预定的内存区域映射到网络上的其他计算机。当任一节点向这块内存区域写入数据时，数据会被自动复制（或反射）到网络上其他节点的对应内存区域中，从而实现数据的实时共享。这种方式的优点在于数据更新几乎无延迟，可以达到微秒级的同步速度，非常适合对时间敏感的应用场景。

### 5.4.1 “生产者/消费者”模式

在试验过程中，数据采集系统软件需同步处理时钟、采集、处理及写入数据等多项任务。若采取顺序处理，不仅会延长读取下一板卡数据的时间，还可能阻塞整个系统。为解决这一问题，本文采用“生产者/消费者”模式来处理线程间的数据传递。

“生产者/消费者”模式被广泛应用于任务分配和资源共享，生产者负责生成数据或消息并将其放入队列，而消费者则从队列中取出这些数据或消息进行处理<sup>[42,43]</sup>。这种模式通过队列实现了生产者和消费者之间的解耦，让软件结构更加清晰，同时队列起到缓冲池的作用，允许生产者和消费者并行操作，提高程序的运行效率。如图 5-5 所示为“生产者/消费者”模式的后面板程序框图示例。

在采集任务启动后，程序会依据输入的通道配置信息，逐一判断每块板卡是否分配有采集任务。一旦发现某块板卡上存在被选中的采集通道，程序将专

#### (2) 数据处理模块

在工业现场，物理环境的变化（如温度、湿度、噪声等）可能影响传感器或测量设备的性能，从而导致数据不准确。此外，在数据传输或存储过程中，网络问题、设备故障以及信号干扰等因素也可能导致数据丢失或畸变。因此，有必要对采集到的传感器数据进行处理。为了确保数据采集的实时性，数据采集系统中的数据处理模块主要负责执行缺失值填补和去噪等操作。这些处理步骤不仅在数据采集初期确保了数据的完整性和真实性，还能使传感器数据实时展示给用户，以供查阅。

考虑到试验台上传感器采集到的数据具有连续性和稳定性，在数据处理模块设计实现了前向填充和线性插值法对传感器数据中存在的缺失值进行填补。如果某个位置的数据缺失，前向填充法会寻找这个缺失值前面的最近一个非缺失值，并将其填入到当前缺失的位置。而线性插值法则是假设两个已知数据点之间的关系是线性的，并根据这两个点的值推算出缺失点的值。缺失值填充过程示例如图 5-11 所示。

#### (1) 表示层 UI

表示层是用户与系统之间交互的核心部分，它负责接收用户的输入并将结果呈现给用户，同时与业务逻辑层交互，以确保用户请求得到正确处理。

表示层的设计以用户体验为中心，确保界面直观、易用、响应迅速。考虑不同用户的需求，提供良好的导航和可访问性。本文的控制系统属于桌面应用，采用 Windows Forms 来开发系统的表示层。采用 MVC 模式将表示层分为三部分：模型（Model）表示数据和业务逻辑，视图（View）负责显示，控制器（Controller）处理用户输入并更新模型和视图。这种设计模式可以减少代码耦合，增强系统的可维护性。在视图中设计按钮、输入框、下拉菜单和复选框等交互控件，方便用户操作系统。将这些 UI 控件与业务逻辑层的数据模型绑定，实现从业务逻辑层或数据源动态加载数据，并在界面上进行展示。通过实现用户输入表单，用户可以提交数据到业务逻辑层进行处理。此过程中包括对用户输入的验证（客户端和服务器端）以及对提交结果的处理。此外，还需要将 UI

控件的事件（如按钮点击、列表选择）绑定到相应的处理函数，以便将用户操作传递给业务逻辑层。

#### (2) 业务逻辑层 BLL

业务逻辑层负责处理应用程序的核心功能和业务规则，承担控制软件主要业务逻辑的任务。位于数据访问层和表示层之间，充当了连接前端用户界面与后端数据管理的桥梁。

在控制系统软件中，业务逻辑层 BLL 的封装和管理试验台对应设备的操作业务逻辑。它从表示层接收用户输入的数据，执行必要的业务操作，然后与数据访问层交互以获取或更新数据。

在设计控制系统时，首先需要明确系统的需求。系统需求包括设备控制、状态监控、报警处理、数据记录等核心功能。此外，业务规则也需要详细定义，如设备工作状态的切换条件、报警条件以及自动化逻辑等内容。控制系统通常需要实时响应外部事件，因此，业务逻辑层必须满足实时处理的要求。同时，控制系统的业务逻辑必须具备高度的安全性和可靠性，能够有效处理异常情况，防止系统失控。

基于以上分析，本文定义了控制系统的关健业务实体和他们之间的关系，主要包括设备、命令、状态和事件。设备主要包括试验台的传感器、泵、阀门等；命令指的是用户可以对试验台设备进行的操作，例如启动、停止、调节参数等；状态表示的是试验台设备或者系统的当前状态，如泵的运行和停止、传感器参数报警等；事件表示系统中发生的事件，如传感器故障、设备参数超限等等。

以设备为例，本文分别实现试验台单个设备的业务操作类，然后通过调用测试项目中关联设备的业务操作类，按照测试流程和逻辑实现相应的测试项目类。最后根据自动化测试流程，将一个或多个测试项目组合成一套自动化测试流程，提高控制系统自动化程度，提高测试效率。这样设计可以保证业务逻辑的复用性和可维护性，当试验台设备更新换代以后，只需要修改对应更换的设备操作业务类即可。并且，每个设备操作业务类只需要完成对应设备的操作功能，减少和其他操作之间的耦合性。业务逻辑层结构如图 5-14 所示。

#### (3) 数据访问层 DAL

数据访问层负责与数据库或其他数据存储介质的交互。数据访问层的主要功能是封装与数据源的所有交互细节，提供一种抽象的、统一的接口，使得应用程序可以通过它来执行数据的读取、写入、更新和删除等操作，而不需要关心底层的实现细节。

本文利用 DAO (Data Access Object) 模式设计了数据访问层 DAL，封装了数据库的访问逻辑，将数据库操作方法抽象为一组接口或方法，使得业务逻辑层只需通过这些接口与数据库交互，而不需要了解底层的数据库结构和访问方式。通过使用独立的执行线程，采取无限循环机制主动从内存反射卡或缓存中提取待处理的试验数据，并将其写入数据库中。

用户可通过数据管理界面类向试验项目管理类、参数配置管理类、通道配置管理类及日志数据管理类发起操作请求，包括但不限于查询试验项目、参数的增删查改以及日志记录的检索等。这些请求由指定的管理类负责处理，并从数据库中检索或更新数据。用户的每次操作，日志数据管理类都会详细记录每个操作的迹象，包括操作者的 ID、操作时间以及操作的具体内容。这一机制不仅为系统误操作和异常行为的追溯提供了可靠的数据支持，还有助于有效追踪潜在的恶意行为。完成日志记录后，操作结果将返回至数据管理界面，以供用户进行查询和审查。

### 6.1.2 采样精度测试

本文主要验证数据采集系统中模拟量测量的采样精度指标是否符合设计要求。由于数据采集模块中数据采集板卡在硬件上大致相同，测试结果具有一定的普适性，因此只对数据采集板卡某一通道的采样精度测试进行介绍。本小节主要对数据采集系统测试环境中位于 2 槽位的数据采集板卡的第 1 通道进行采样精度测试。

在采样精度测试之前，对数据采集板卡进行精度校准是必不可少的一个环节。为了减少数据采集板卡在制作完成时存在的系统误差带来的影响，让测量设备的测试结果更加接近真实值，以符合系统预期的设计指标。首先进行偏移校准，为了保证对偏移电压测试的准确性，将数据采集板卡中模数转换芯片（ADC）的降采样抽取率设置为最高，并对偏移电压进行多次测量后取平均值；同时考虑数据采集板卡电子元器件内部的热稳定性对偏移校准产生的影响，在设备通电正常运行 30 分钟以上进行测量。

# 第七章 总结与展望

## 7.1 工作总结

本文针对试验台被试件生产厂家在产品测试中存在的问题，设计并实现了一套飞机燃油系统半物理试验台测控系统。有效解决企业在产品测试过程中存在的测试成本高、自动化程度低、扩展性差、测试型号单一等问题。同时本文采用 MATLAB 构建基于 LSTM 模型预测试验台设备状态，使技术人员掌握被试验台设备状态变化趋势，避免试验台设备出现过载运行，防止设备损坏并预防安全事故发生。本文主要完成工作如下。

(1) 根据国家标准和企业实际需求，对系统进行需求分析，设计系统总体架构和划分功能模块，最终以 PXI+工控机的架构完成一套飞机燃油系统半物理试验台测控系统。实现了对试验台设备的状态监测和控制。

(2) 完成测控系统硬件结构的设计。明确测控系统的采集与控制对象，将测控系统分为数据采集系统和控制系统两部分，并对 PXI 控制器、工控机、数据采集卡等关键设备选型，分析测控系统需要采集和控制的参数得出数据采集与控制列表，根据数据采集与控制列表对采集板卡的采集通道的进行分配。

(3) 完成系统软件的设计与开发。测控系统的软件包括数据采集系统软件和控制系统软件，这两部分使用不同的开发语言进行设计与实现。数据采集系统基于 LabVIEW 进行开发，根据数据采集和控制需求实现系统的数据采集、控制输出、数据处理等功能模块。控制系统软件基于.NET Framework4.0 框架开发，分为界面层、业务逻辑层、数据存储层。本文详细概述各层组件的使用以及用户管理模块、系统配置模块、产品试验模块、设备管理模块、辅助模块的业务逻辑和设计过程和数据的管理方案。

(4) 对系统模块和整体运行进行实际测试，包括系统的现场部署和调试。并且，对各个系统组件的运行性能进行测试，展示系统的运作效果。

综上所述，本文所设计的飞机燃油系统半物理试验台测控系统，成功实现了对试验台设备状态的监测与控制。帮助企业控制试验台进行更加准确和高效的试验，为企业对被试件的研发提供了有力的技术支持。

## 7.2 工作展望

经过设计与实现，飞机燃油系统半物理试验台的测控系统已经取得了一定的成果与试验报告。然而，考虑到技术的持续进步与需求的不断升级，有必要在现有基础上，进一步研发一个更全面、自动化及智能化水平更高的测控系统。针对软件性能、功能扩展及软件同步功能等方面，该系统仍有待优化与提升。具体而言，可以从以下几个方面着手进行改进：

(1) 提升数据采集系统软件的性能表现。当前，数据采集系统在完成试验台测试任务时，CPU 的使用率维持在约 75%。为了在不改变数据采集计算机硬件配置的条件下，通过对代码的精细优化来提升程序的执行效率，进一步提高系统的实时性。

(2) 扩展测控系统软件的 Web 功能。为了满足更广泛用户的访问需求，并提升数据共享与协作的便捷性，计划将测控系统软件通过 Web 方式扩展至网页端。允许用户，实现对试验台设备状态的远程监控、试验项目的高效管理。
